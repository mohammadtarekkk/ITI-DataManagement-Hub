# LVM (Logical Volume Manager)

Allows **dynamic resizing** and **pooling** of multiple physical disks into flexible volumes.

## Components
```
Physical Volumes (PV)  →  Volume Group (VG)  →  Logical Volumes (LV)
  /dev/sdb, /dev/sdc        my_vg                 my_lv
```
1. **PV**: Raw disk/partition.
2. **VG**: Pool of PVs.
3. **LV**: Virtual partition carved from a VG.

## Creation Workflow
1. `pvcreate /dev/sdb /dev/sdc` — initialize disks.
2. `vgcreate my_vg /dev/sdb /dev/sdc` — create pool.
3. `lvcreate -L 500M -n my_lv my_vg` — create volume.
4. `mkfs.ext4 /dev/my_vg/my_lv` — format.

## Management
| Action | Command |
|--------|---------|
| Extend LV | `lvextend -r -L +200M /dev/my_vg/my_lv` |
| Extend VG | `vgextend my_vg /dev/sdd` |
| Reduce LV (ext4 only) | `lvreduce -r -L -100M /dev/my_vg/my_lv` (requires unmount) |
| Remove | `lvremove` → `vgremove` → `pvremove` |

> [!NOTE]
> The `-r` flag on `lvextend` / `lvreduce` automatically resizes the filesystem too.
