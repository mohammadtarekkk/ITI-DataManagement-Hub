# Schema Objects

Beyond tables, a schema can contain several other structural objects.

## 1. Sequences (Auto Increment)
Generates consecutive numeric values. Older Oracle versions rely on this for Primary Keys.
```sql
CREATE SEQUENCE dept_seq 
START WITH 370 
INCREMENT BY 10;

INSERT INTO departments (dept_id, name) VALUES (dept_seq.NEXTVAL, 'Java');
```

## 2. Identity Columns (Oracle 12c+)
Replaces external sequences. The table automatically manages the auto-increment.
```sql
-- Generates value unconditionally
CREATE TABLE BOOKS2 (
    BOOK_ID NUMBER GENERATED ALWAYS AS IDENTITY,
    BOOK_NAME VARCHAR2(250)
);

-- Generates value ONLY if NULL is passed
CREATE TABLE BOOKS3 (
    BOOK_ID NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY,
    BOOK_NAME VARCHAR2(250)
);
```

## 3. Synonyms
Creates an alias or an alternative name for a schema object (useful for hiding schema ownership).
```sql
-- Private synonym (accessible only to you)
CREATE SYNONYM my_sales FOR hr.sales_data;

-- Public synonym (requires DBA rights, available to all)
CREATE PUBLIC SYNONYM public_locations FOR hr.locations;
```
Now querying `my_sales` automatically translates to `hr.sales_data`.

## 4. Views
A logical table based on a SELECT query. It doesn't store data, just the query definition.
- Simplifies complex queries.
- Enhances security by hiding specific columns or rows.
